Customization

  • app.css
    Here you can determine what stylings will be in your project. By default all of Foundation is imported, but you can deactivate and select specific styles you want to import.
  • _settings.scss
    Here you can activate preselected customization variables and customize Sass where it will be compiled into CSS.
  • stylesheets folder
    We include a stylesheets folder where you can customize your CSS. This files has basic Foundation CSS styling and can be customized here. Add any additional css style sheets in this folder.

JavaScript

  • app.js
    This is where you should put your javascript plugins or custom scripts to be imported into your index file. We do this to keep the markup on the html clean and simple by keeping your JS in a separate place.
  • jquery.min.js
    A minified jQuery 2 file which is required for some Foundation JavaScript to work.

Bower Components

  • The recommended install uses Bower to package up all the Foundation components and manage any updates. Any changes made to files in this folder will be overwritten when your file is compiled.
  • fastclick.js
    We’ve integrated fastclick.js so you have the option to give your mobile users have a snappier experience.
  • foundation folder
    This is where we keep all the files that make foundation work. Any CSS and SCSS partials and javascript live here. Including the components here allows you to easily update your project with ```bower update``` command.
  • jquery
    We've included jQuery 2.0 in here because anytime there is an update to jQuery with a Foundation release it will be updated as well.
  • jquery.cookie
    This is included in Foundation for our Joyride feature. It allows the browser to create a cookie specifically for the option to not show the Joyride tour a second time.
  • modernizr
    This acts as a shim for HTML5 elements that older browsers may not recognize, and provides detection for mobile devices:
  • normalize.scss
    We've included a copy of Normalize here to give you the option to remove quirks in browsers so that all webpages render elements consistently.

Pre-Packaged Essentials

  • index.html
    This is an example index file which contains some sample markup (the grid, buttons, etc) as well as the basic structure of a responsive Foundation page. Feel free to change or destroy this file as you see fit, just bear in mind that some of the document `````` is required for the page to work, and the CSS/JS is linked up at the end of the document.
  • humans.txt
    A basic humans.txt file you can use as a basis for your site, if you'd like to have a humans.txt file
  • robots.txt
    Another basic starter file, this is a sample robot file you can amend and modify to suit your particular crawling needs.
  • bower.json
    During updates, this the file you can specify what version to update to.
  • config.rb
    This is where Ruby is told what assets to assemble and how to map your project's file locations.
