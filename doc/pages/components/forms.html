--- layout: doc/layouts/default.html --- # Forms

With Foundation, we set out to create an easy to handle, powerful and versatile form layout system. A combination of form styles and the Foundation grid means you can do basically anything.

*** ### Building Forms with HTML Ceating a form in Foundation is designed to be easy but extremely flexible. Forms are built with a combination of standard form elements, as well as the Grid (rows and columns). Form elements in Foundation are styled based on their type attribute rather than a class, and can be sized in a couple of ways: * You can size inputs using column sizes, like `.large-6`, `.small-6`. * You can create row elements inside your form and use columns for the form, including inputs, labels and more. Rows inside a form inherit some special padding to even up input spacing. This is an example form we've created that is laid out using the grid:

*** ### Building with Mixins A lot of elements in Foundation can be created using SCSS mixins so you can use your own classes (or no classes at all). To use these mixins, you'll need to have the extension installed or grab [_variables.scss](https://github.com/zurb/foundation/blob/master/scss/foundation/_variables.scss), [_forms.scss](https://github.com/zurb/foundation/blob/master/scss/foundation/components/_forms.scss), _global.scss and [_buttons.scss](https://github.com/zurb/foundation/blob/master/scss/foundation/components/_buttons.scss) from Github and throw them into a Foundation folder in your project directory. From there, you can import the files at the top of your own SCSS stylesheet, like so: {{#markdown}} ```scss @import "foundation/variables", "foundation/components/global", "foundation/components/buttons", "foundation/components/forms"; ``` {{/markdown}} We include most of the form styles by default, but the ones that are attached to HTML classes can be accessed via the mixins below.
