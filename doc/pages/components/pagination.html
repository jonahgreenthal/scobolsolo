--- layout: doc/layouts/default.html --- # Pagination

Moving between pages has become less common with the advent of longer pages and AJAX loading, but it can still be useful for long, repetitive listings or content.

{{> examples_pagination_basic}} *** ### Basic Add a `.pagination` class to a `ul` to get started, then add list items with links in them. * Create arrows by adding an `.arrow` class to the first and last list items. * To mark the current page by adding a `.current` class to a list item. * Add a class of `.unavailable` to a list item to make it not clickable, like for the ellipsis in the middle.

*** ### Build with a Mixin We've included SCSS mixins used to style pagination. To use the mixin, you'll need to have the extension installed or grab _variables.scss, _global.scss and [_pagination.scss](https://github.com/zurb/foundation/blob/master/scss/foundation/components/_pagination.scss) from Github and throw them into a Foundation folder in your project directory. From there, you can import the files at the top of your own SCSS stylesheet, like so: {{#markdown}} ### SCSS ```scss @import "foundation/variables"; @import "foundation/components/global"; @import "foundation/components/pagination"; ``` {{/markdown}} If you are using the mixins, you may include the styles on whatever class or ID you'd like to the unordered list. The interior classes come with the mixin, just make sure you follow our markup structure: {{#markdown}} ### HTML ```html {{> examples_pagination_mixin}} ``` {{/markdown}} ##### Quick Mixin You can build your pagination using our global mixin by including it on your own custom class. The mixin contains all the necessary styles for positioning and styling the unordered full list of items, the code looks like this: {{#markdown}} ```scss .your-class-name { @include pagination; } ``` {{/markdown}}
