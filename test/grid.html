The Grid

Create powerful multi-device layouts quickly and easily with the 12-column, mobile first, nestable Foundation 4 grid. If you're familiar with grid systems, you'll feel right at home.

Basic Usage

The grid can be used to create any number of simple or complex layouts that respond to our single large screen breakpoint. You can use the default stacking column grid that you are used to from previous version of Foundation, as well as, a full fleshed out small and large grid for creating responsive layouts across devices. We also removed the .column/.columns class as a class chain to simplify markup and styles.

Since the grid is built mobile first, it used media queries to serve the large layouts. This means that any browser that doesn't support media queries will default to the stacking columns, including IE8.

Using the Small Grid

We also include a small grid that can be used in conjunction with the large and stacking column grid to create very complex layouts that work for the viewport you need. To use this grid you simply include another class like, .small-#. If you only use a small column class, your large screen layout will inherit stacking columns, so make sure to include a large grid if you don't want this. You can even declare a different number of columns for small and large.

Note: View this page on a small device or shrink your browser to see the small grid in action.

.small-4

Floating the Last Column

In order to work around browsers' different rounding behaviours, Foundation will float the last column in a row to the right so the edge aligns. If your row doesn't have a count that adds up to 12 columns, you can tag the last column with class="end" in order to override that behaviour.

Nesting Columns

The grid allows for nesting down as far as you'd like, though at a certain point it will get absurd. You can use this nesting to create quite complex layouts, as well as some other tricks like form layouts or visual elements. In order to nest properly you you must have you markup structured with a row and a column, then a row inside of the column. You can't have a row followed by a row that isn't inside of a column.

Offsetting Columns

Offsets allow you to create additional space between columns in a row. The offsets run from .large-offset-1 all the way up to .large.offset-11. Like the rest of the grid, they're nestable.

Centering Columns

Centered columns are placed in the middle of the row. This does not center their content, but centers the grid element itself. This is a convenient way to make sure a block is centered, even if you change the number of columns it contains. Note: There cannot be any other column blocks in the row for this to work.

.large-2.large-centered
.large-4.large-centered
.large-6.large-centered

Centering Small Columns

As with everything else, we've included the ability to center small columns as well. This can be used in conjunction with large centered columns and you can even combine different sized columns.

Note: View this page on a small device or shrink your browser to see the small grid in action.

.small-8.large-4.small-centered.large-centered
.small-4.large-8.small-centered.large-centered

Source Ordering Columns

Sometimes within the grid you want the order of your markup to not necessarily be the same as the order items are flowed into the grid. Using our source ordering classes, .large-push-#/.large-pull-#, you can shift columns around on desktops and tablets. On phones, the grid will shift back to normal unless you also declare .small-push-#/.small-pull-# classes. You can create different source orders for each grid as well by combining them with the large and small grid like below.

.large-4.large-push-8
.large-8.large-pull-4
.small-8.large-8.large-push-4
.small-4.large-4.large-pull-8
